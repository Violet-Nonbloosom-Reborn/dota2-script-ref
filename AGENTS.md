# AGENTS.md

## Package Manager

Use **pnpm** (not npm). The `pnpm-workspace.yaml` configures build script approvals.

## Commands

```bash
pnpm install              # Install dependencies
pnpm run build            # Full build: clean → extract → process-dump
pnpm run build:extract    # Extract from Dota 2 VPKs (requires local Dota 2 install)
pnpm run build:process-dump # Process dumper/dump file only (no Dota 2 needed)
pnpm run auto-dump        # Launch Dota 2 and dump API (requires Dota 2 + tools)
pnpm run test             # Jest tests
pnpm run lint             # ESLint + Prettier check
pnpm run fix:prettier     # Auto-fix formatting
```

## Project Structure

| Directory | Purpose |
|-----------|---------|
| `dumper/` | Lua scripts + TS launcher that dumps Dota 2 API via vConsole |
| `dumper/dump` | Raw dump output (git-tracked, MediaWiki table format) |
| `build/` | TypeScript scripts that parse dump → structured JSON |
| `skills/` | Agent Skill definition and generated reference files |
| `skills/reference/` | Generated JSON reference files (git-ignored) |
| `test/` | Jest tests |

## Dump Format

The `dumper/dump` file contains multiple sections, each started by `$> command_name`:

| Section | Format | Content |
|---------|--------|---------|
| `dump_panorama_css_properties` | Custom text | Panorama CSS properties |
| `dump_panorama_events` | MediaWiki table | Panorama custom events |
| `cl_panorama_script_help_2` | MediaWiki table | Panorama enums + API classes (with params, no return types) |
| `cl_panorama_typescript_declarations` | TypeScript interface | Panorama API return types (supplement) |
| `script_reload` | JSON | VScripts Lua API (server) |
| `cl_script_reload` | JSON | VScripts Lua API (client) |

### MediaWiki Table Format

```
=== ClassName ===
{| class="standard-table"
! Function ! Signature ! Description
|-
| MethodName | <code>Class.Method( type arg )</code> | Description |
|}
```

For enums:
```
=== EnumName ===
{| class="standard-table"
! Enumerator ! Value ! Description
|-
| EnumName.MemberName | 0 | Description |
|}
```

## Build Flow

1. `build:extract` — Reads VPK files from Dota 2 install, generates events/enums
2. `build:process-dump` — Parses `dumper/dump`, generates `skills/reference/**/*.json`

For skill generation, only `build:process-dump` is needed (no VPK access).

## Output Files

```
skills/reference/
├── panorama/
│   ├── api.json        # Panorama JS API classes (23 namespaces)
│   ├── css.json        # Panorama CSS properties
│   ├── enums.json      # Panorama enums (48 enums)
│   └── events.json     # Panorama custom events
└── vscripts/
    ├── api.json        # VScripts Lua API (classes, functions)
    ├── api-types.json  # VScripts type declarations
    └── enums.json      # VScripts enums and constants
```

## Panorama API Generation

The Panorama API is generated by merging two data sources:

1. **Primary**: `cl_panorama_script_help_2` — provides class list, method names, parameters
2. **Supplement**: `cl_panorama_typescript_declarations` — provides return types

Classes not found in `typescript_declarations` will have `returns: "any"`.
